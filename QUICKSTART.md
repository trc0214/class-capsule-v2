# 🚀 快速開始指南 (5 分鐘上手)

## 1️⃣ 獲取 API 密鑰 (5 分鐘)

> 詳細環境變量管理方式見 [ENV.md](ENV.md)

### Azure Speech Service 密鑰

```
1. 訪問 https://portal.azure.com
2. 創建資源 → 搜索 "Speech"
3. 點擊 "Create"
   - Region: japaneast (日本東部)
   - Pricing tier: Standard
4. 創建完成後，複製 Key1 或 Key2
```

> ⚠️ 注意: Free tier 不支持實時流式識別，必須使用 Standard

### Google Gemini API 密鑰

```
1. 訪問 https://aistudio.google.com
2. 點擊"Get API Key"
3. 點擊"Create API Key in new project"
4. 複製顯示的 API 密鑰
```

## 2️⃣ 打開應用

### 選項 A: 線上使用（推薦）

訪問: https://class-capsule-v2.vercel.app

> 如果還未部署，繼續使用選項 B

### 選項 B: 本地運行

```bash
# 方法 1: Python
cd class-capsule-v2
python -m http.server 8000

# 方法 2: Node.js
npx http-server

# 然後打開: http://localhost:8000
```

## 3️⃣ 首次配置 (1 分鐘)

```
1. 頁面頂部找到「⚙️ 設定您的 API 密鑰」
2. 貼上 Azure Speech API 密鑰
3. 貼上 Gemini API 密鑰
4. 點擊「💾 保存設定」
```

✅ 密鑰會安全地保存在您的瀏覽器中，不會發送給任何服務器

## 4️⃣ 開始錄音 (1 分鐘)

```
1. 點擊「⏺️ 開始錄音」
2. 允許瀏覽器訪問麥克風
3. 開始講話，文字會實時出現
4. 完成後點擊「⏹️ 停止錄音」
```

## 5️⃣ 生成摘要 (1 分鐘)

```
1. 選擇摘要風格 (推薦「結構化筆記」)
2. 選擇摘要長度 (推薦「中等 1000 字」)
3. 點擊「🤖 生成 Gemini AI 摘要」
4. 等待 10-30 秒，AI 會生成結構化筆記
```

## 🎯 常用操作

### 複製轉錄文本
```
轉錄面板 → 點擊「📋 複製轉錄文本」
```

### 下載文件
```
轉錄: 點擊「⬇️ 下載轉錄 (TXT)」
摘要: 點擊「⬇️ 下載摘要 (Markdown)」
```

### 編輯摘要內容
```
右擊摘要文本 → 全選 → 複製到 Word/Notion
```

### 重新開始
```
點擊「🔄 重置」，清除所有轉錄和摘要
```

## ❓ 常見問題速查

### Q: 「無法訪問麥克風」

**A:** 檢查：
- [ ] 麥克風線路是否連接
- [ ] 瀏覽器是否在隱私模式
- [ ] 在設置中允許網站訪問麥克風

**解決方法:**
```
Chrome: 設置 → 隱私 → 網站設置 → 麥克風 → 允許
Firefox: 地址欄左側 → 信息圖標 → 允許麥克風
```

### Q: 「Azure API 錯誤」

**A:** 檢查：
- [ ] API 密鑰是否正確複製
- [ ] 是否使用了 Free tier（必須要 Standard）
- [ ] 區域是否是 japaneast
- [ ] Azure 訂閱是否還有配額

### Q: 「Gemini 摘要生成很慢」

**A:**
- 第一次通常需要 20-30 秒
- 網絡延遲可能導致更長時間
- 文本太長（10000+ 字）會更慢
- 等待幾秒，如果一直卡著，刷新頁面重試

### Q: 「識別中斷了」

**A:**
- 系統會自動重新連接（最多 5 次）
- 如果失敗，點擊「停止」再「開始」重新開始
- 確保網絡連接穩定

### Q: 「密鑰被清除了」

**A:**
- 清除瀏覽器緩存時會清除本地密鑰
- 使用無痕模式不會保存密鑰
- 解決: 重新輸入密鑰即可
